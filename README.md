# 风信来信：风信邮筒

一个基于原生 `HTML + CSS + JavaScript + Canvas 2D` 制作的互动叙事小游戏。

玩家会在校园回忆中做出选择，挑选回信对象、补完没有说出口的话，最后根据过程中的情感倾向与共鸣度，收到一张属于自己的“回信卡”。

## 项目特点

- 纯前端实现，无需构建工具
- 使用单个 `canvas` 完成主要界面与交互绘制
- 支持角色选择，并联动后续名字、文本、人称和对话框配色
- 支持共鸣度结算、多结局分支与回信卡导出
- 支持全局 BGM、重置、主角改名
- 针对手机端做了额外适配
  - 保存回信卡支持系统分享或长按保存
  - 主角改名使用页面内弹层，不依赖浏览器原生 `prompt`

## 在线体验方式

这是一个静态前端项目，直接部署到任意静态托管平台即可运行，例如：

- GitHub Pages
- Vercel
- Netlify
- 任意支持静态文件托管的服务器

## 本地运行

### 方式一：直接打开

直接用浏览器打开 `index.html` 即可运行。

### 方式二：本地静态服务器

如果你希望更接近正式部署环境，可以在项目根目录启动一个静态服务器，例如：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000/
```

## 操作说明

- 标题页可开始游戏或修改主角名字
- 进入同学选择页后，第一次点击角色卡会高亮，第二次点击才确认
- 校园剧情中的选项会影响情感属性与最终结局
- 写信阶段需要选择 2-3 句没说出口的话
- 折信阶段需要拖动纸角完成折叠
- 投递阶段需要在顺风时点击信封
- 结算页可保存回信卡
  - 电脑端支持直接保存为 `PNG` 或 `JPG`
  - 手机端优先调用系统分享，不支持时可长按预览图保存

## 共鸣度说明

当前版本的共鸣度会受到以下因素影响：

- 好感
- 真诚
- 遗憾
- 距离
- 风顺程度

其中：

- 好感、真诚越高，共鸣度越高
- 遗憾、距离越高，共鸣度越低
- 风顺分会带来加成，但影响小于前几项核心属性
- 风顺分超过 `80` 后按 `80` 计算
